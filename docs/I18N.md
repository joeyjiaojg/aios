# Internationalization (i18n)

> AIOS is designed from the ground up with full internationalization support.

## Goals

- **Unicode-first**: All kernel strings are UTF-8 encoded
- **Locale-aware**: System supports per-user language and region settings
- **Input flexibility**: Multiple keyboard layouts and input methods
- **Font support**: Unicode font rendering for all scripts

## Architecture

### Kernel-Level i18n
- UTF-8 string handling in kernel
- Locale data loaded at boot from filesystem
- Unicode console output

### Userspace i18n
- `setlocale()` compatible API
- `gettext()`-style translation framework
- Unicode-aware libc functions
- Font rendering pipeline

### Input Methods
- Keyboard layout tables
- IME (Input Method Editor) support for CJK
- Compose key sequences

### Display
- Unicode font loading (BDF/PCF/TTF)
- Right-to-left (RTL) text rendering
- Complex script shaping (HarfBuzz-style)

## Contributing Translations

Translation files will be stored in `locales/` directory with the following structure:

```
locales/
├── en-US/
│   ├── kernel.po
│   └── userspace.po
├── zh-CN/
│   ├── kernel.po
│   └── userspace.po
└── ...
```

---

> AI will automatically generate translation drafts, but human review is recommended for accuracy.
